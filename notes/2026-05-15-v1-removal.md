# v1 removal notes

The v1 transport retirement removes the legacy browser WebSocket surface and
the object-scoped REST SSE stream.

## Integration-visible changes

- `GET /ws` is gone. Browser and agent live delivery use the v2 turn network.
- `GET /api/objects/{id}/stream` now returns `410 E_GONE`. Durable catch-up is
  through `GET /api/objects/{id}/log`; live frames are delivered through the v2
  browser turn network.
- REST verb calls are executed through the v2 turn machinery so the same commit
  authority and transcript validation path is used for browser and REST ingress.

Custom integrations that consumed object SSE streams need to switch to `/log`
polling for durable backfill or maintain a v2 turn-network connection for live
delivery.
